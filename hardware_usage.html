Hardware Usage¶

Generally you need for actions to deal with networks of spiking neurons. Set the parameters of the neurons, connect it to other neurons, stimulate & monitor, analyze. This section will explain you how to do these using our python interface.

  • Setting up the parameters: ex: ifslwta1.bias.pinj.v=2.86
  • Addressing: This part is slightly unintuitive: There are 2 types of addresses which uniquely describes each neuron.
  1. ‘Human readable address’ corresponding to “Synapse Y of Neuron X”, noted [[X],[Y]] in python
  2. Hardware addresses corresponding to decimal representations of bits. Basically what the hardware understands, and you don’t.

The pyST tool allows you to translate between the two. Other operations are performed by the following python functions:

  • Mapping: From the pyAex package, ex: setmappings(mappings), See setmappings
  • Stimulation & Monitoring: From the pyAex package, ex: STAEXIO(stStim)
  • Spike train visualization and analysis: from the pyST package, composite_plot(stOut)

Powering up the AEX boards¶

There is a particular power cycle that one must respect:

  1. Unplug the USB cables from the AEX boards
  2. Power up the AEX boards
  3. Wait until the red LED has turned off then plug the USB cable.
  4. Turn on the AMDA boards
  5. Plug in the USB cables, starting with the AEX board you wish to monitor from (the devices numbers are assigned when the USB cable is plugged in)
  6. (Optional) In a setup with a mapper, only the USB cable of the AEX board which is used for monirtoring must be connected, the others can be removed (this saves USB cables).
  7. Set the biases on the chip

That should do it. If you have problems ask Emre or Daniel
